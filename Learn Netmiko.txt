         Golden rule for Multi-Device Automation
         1.Always wrap each device in its own try/except block
            one failure should not stop the whole script
         2.Always print which device connecting to
            make debugging easy
         3.Always save output per device seprately
            Easy to trace back result
         4.Always print summary at end
            to know what succeeded and what failed
         5.Always test script on one devic first, then scale to many

6. Lesson6 : Parsing CLI output with TextFSM & NTC Templats
   #Why raw CLI output is hard to work with
     when we run command "send_command()" we get back a raw string, from that output
     to get just IP address of any interface we have to split by space, split by lines.
     That messy and break easily . TextFSM fixes this.
   #What is TextFSM
     TextFSM is python library that takes raw output and turn it into
     interface ip status[gig0/0 1.1.1.1 up]
   #What is NTC Templates?
     writing TextFSM rules ourself is complex, NTC Templates is a free library that has
     pre-written rules for hundreds of cisco/juniper/arista commands already done for us.
